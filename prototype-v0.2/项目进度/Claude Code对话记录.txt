> # 项目名称：3D 模型生成智能体

  ## 项目目标
  开发一个 Web 应用，用户输入文字描述或上传图片，调用 AI 服务生成 3D 模型，并在网页中实时预览、拖拽查看模型。

  ## 技术栈
  - 后端：Node.js + Express
  - 前端：HTML/CSS/JavaScript
  - 3D 渲染：Three.js (r128)
  - API 调用：
    - DeepSeek（优化提示词）
    - Meshy（生成 3D 模型）

  ## 功能需求
  1. **文本生成 3D**
     - 用户输入英文或中文描述
     - 调用 DeepSeek API 优化为英文提示词
     - 调用 Meshy API 创建 3D 模型任务
     - 轮询任务状态，完成后显示模型

  2. **图片生成 3D**（可选）
     - 用户上传图片
     - 调用 Meshy 的 image-to-3d API
     - 轮询任务状态，完成后显示模型

  3. **3D 预览**
     - 使用 Three.js 加载 GLB 模型
     - 支持鼠标拖拽旋转、滚轮缩放
     - 显示网格、灯光控制

  4. **任务状态反馈**
     - 显示进度条和状态文本
     - 任务完成后提供下载链接

  ## 文件结构
  项目应包含以下文件：
  - server.js          # 后端主文件
  - index.html         # 主页面
  - style.css          # 样式
  - script.js          # 前端逻辑
  - .env               # 环境变量（存储 API 密钥和端口）
  - package.json       # 依赖配置

  ## API 密钥与端口
  - 使用环境变量管理密钥，不得硬编码。
  - 端口：3000（可通过 .env 配置）
  - Meshy API Key: `YOUR_MESHY_API_KEY`
  - DeepSeek API Key: `YOUR_DEEPSEEK_API_KEY`

 Plan: 3D Model Generation Web App

 Context

 Build a new web application from scratch that lets users generate 3D models from text descriptions or images using AI
 APIs (DeepSeek for prompt optimization, Meshy for 3D generation), with Three.js-based 3D preview.

 Files to Create (6 files)

 1. package.json

 - Dependencies: express, dotenv, cors, multer (for image upload), node-fetch (for API calls)

 2. .env

 - MESHY_API_KEY, DEEPSEEK_API_KEY, PORT=3000

 3. server.js — Express backend

 - POST /api/generate-text — Text-to-3D flow:
   a. Receive user text description
   b. Call DeepSeek API (https://api.deepseek.com/chat/completions) to optimize the description into an English 3D
 modeling prompt
   c. Call Meshy API (https://api.meshy.ai/openapi/v2/text-to-3d) to create a task
   d. Return task ID
 - POST /api/generate-image — Image-to-3D flow:
   a. Receive uploaded image via multer
   b. Convert to base64, call Meshy image-to-3d API
   c. Return task ID
 - GET /api/task/:id — Poll task status:
   a. Call Meshy API to get task status/progress
   b. Return status, progress percentage, and model URL when complete
 - Serve static files (index.html, style.css, script.js)

 4. index.html — Main page

 - Text input area + "Generate" button
 - Image upload input + "Generate from Image" button
 - Progress bar + status text area
 - Three.js canvas container for 3D preview
 - Download link (hidden until model ready)

 5. style.css — Styling

 - Clean, centered layout
 - Styled input controls, buttons, progress bar
 - Full-width 3D viewer area

 6. script.js — Frontend logic

 - Text generation: POST to /api/generate-text, then poll /api/task/:id
 - Image generation: POST FormData to /api/generate-image, then poll
 - Polling: setInterval checking task status, updating progress bar
 - Three.js setup: Scene, camera, renderer, OrbitControls, lighting
 - GLTFLoader to load the model URL when task completes
 - Download button to fetch the GLB file

 Key API Details

 DeepSeek API

 - Endpoint: https://api.deepseek.com/chat/completions
 - Model: deepseek-chat
 - System prompt instructs it to convert user input to optimized English 3D model description

 Meshy Text-to-3D API

 - Endpoint: https://api.meshy.ai/openapi/v2/text-to-3d
 - POST with { mode: "preview", prompt, art_style: "realistic" }
 - Returns { result: taskId }

 Meshy Task Status API

 - GET https://api.meshy.ai/openapi/v2/text-to-3d/{taskId}
 - Returns { status, progress, model_urls: { glb } }

 Meshy Image-to-3D API

 - POST https://api.meshy.ai/openapi/v2/image-to-3d
 - Body: { image_url: "data:image/png;base64,...", enable_pbr: true }

 Three.js Setup

 - Use CDN imports for Three.js r128, GLTFLoader, OrbitControls
 - Scene with ambient + directional lights
 - OrbitControls for drag rotation and scroll zoom
 - Auto-center and scale model after loading

 Verification

 1. Run npm install then node server.js
 2. Open http://localhost:3000
 3. Enter a text description (e.g. "a red toy car"), click Generate
 4. Verify progress bar updates, model loads in 3D viewer
 5. Test drag rotation and scroll zoom
 6. Test image upload flow
 7. Test download link

     Plan: 3D Model Generation Web App

     Context

     Build a new web application from scratch that lets users generate 3D models from text descriptions or images using AI APIs (DeepSeek for prompt optimization, Meshy for
     3D generation), with Three.js-based 3D preview.

     Files to Create (6 files)

     1. package.json

     - Dependencies: express, dotenv, cors, multer (for image upload), node-fetch (for API calls)

     2. .env

     - MESHY_API_KEY, DEEPSEEK_API_KEY, PORT=3000

     3. server.js — Express backend

     - POST /api/generate-text — Text-to-3D flow:
       a. Receive user text description
       b. Call DeepSeek API (https://api.deepseek.com/chat/completions) to optimize the description into an English 3D modeling prompt
       c. Call Meshy API (https://api.meshy.ai/openapi/v2/text-to-3d) to create a task
       d. Return task ID
     - POST /api/generate-image — Image-to-3D flow:
       a. Receive uploaded image via multer
       b. Convert to base64, call Meshy image-to-3d API
       c. Return task ID
     - GET /api/task/:id — Poll task status:
       a. Call Meshy API to get task status/progress
       b. Return status, progress percentage, and model URL when complete
     - Serve static files (index.html, style.css, script.js)

     4. index.html — Main page

     - Text input area + "Generate" button
     - Image upload input + "Generate from Image" button
     - Progress bar + status text area
     - Three.js canvas container for 3D preview
     - Download link (hidden until model ready)

     5. style.css — Styling

     - Clean, centered layout
     - Styled input controls, buttons, progress bar
     - Full-width 3D viewer area

     6. script.js — Frontend logic

     - Text generation: POST to /api/generate-text, then poll /api/task/:id
     - Image generation: POST FormData to /api/generate-image, then poll
     - Polling: setInterval checking task status, updating progress bar
     - Three.js setup: Scene, camera, renderer, OrbitControls, lighting
     - GLTFLoader to load the model URL when task completes
     - Download button to fetch the GLB file

     Key API Details

     DeepSeek API

     - Endpoint: https://api.deepseek.com/chat/completions
     - Model: deepseek-chat
     - System prompt instructs it to convert user input to optimized English 3D model description

     Meshy Text-to-3D API

     - Endpoint: https://api.meshy.ai/openapi/v2/text-to-3d
     - POST with { mode: "preview", prompt, art_style: "realistic" }
     - Returns { result: taskId }

     Meshy Task Status API

     - GET https://api.meshy.ai/openapi/v2/text-to-3d/{taskId}
     - Returns { status, progress, model_urls: { glb } }

     Meshy Image-to-3D API

     - POST https://api.meshy.ai/openapi/v2/image-to-3d
     - Body: { image_url: "data:image/png;base64,...", enable_pbr: true }

     Three.js Setup

     - Use CDN imports for Three.js r128, GLTFLoader, OrbitControls
     - Scene with ambient + directional lights
     - OrbitControls for drag rotation and scroll zoom
     - Auto-center and scale model after loading

     4. Verify progress bar updates, model loads in 3D viewer
